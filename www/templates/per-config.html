Peripherals

Attached Displays

Show either the moOde WebUI or PeppyMeter on an attached display. Both HDMI and DSI attached displays are supported. There are settings in common between these two types of displays in the General settings section. Specific settings for each type are in their respective sections.

moOde WebUI
$_select[local_display_on] $_select[local_display_off]
$_screen_res_local_display Show the moOde WebUI on a locally attached display.
Enter the the target URL in the format "http://HOST_NAME_OR_IP_ADDRESS/".
Default is "http://localhost/"
CoverView Coverview is: $_coverview_onoff

Peppy Meter/Spectrum
$_select[peppy_display_on] $_select[peppy_display_off]
$_screen_res_peppy_display Show Peppy Meter or Spectrum on a locally attached display.
Note: If Crossfeed, Graphic or Parameteic EQ is on, Peppy cannot be turned on if:
- Playing to a Bluetooth speaker.
- ALSA output mode is set to "Default".
Choose between VU Meter and Spectrum displays.
Peppy Display
Meter/Spectrum settings
$_select[enable_peppyalsa_on] $_select[enable_peppyalsa_off]
Adds PeppyALSA as a standalone driver to the audio chain. This allows user written programs to switch between WebUI and Peppy display using the command: sudo moodeutl --setdisplay [webui | peppy].
$_select[touchmon_svc_on] $_select[touchmon_svc_off]
Displays Peppy when MPD starts playing. Touching the display switches to the WebUI. If a touch does not occur within the Timeout period the display switches back to Peppy.
When the WebUI is displayed and MPD playback starts playback this timeout determines when the display will be switched to Peppy if no touch occurs.

Change between Landscape and Portrait orientation. Assumes Landscape is the native orientation.
$_select[hdmi_cec_on] $_select[hdmi_cec_off]
Automatically turn the display on when moOde starts or when wake on play is initiated. Turn the display off when moOde is shutdown.
Set the CEC version to be used. Older devices may require CEC version 1.4.
$_select[hdmi_enable_4kp60_on] $_select[hdmi_enable_4kp60_off]
By default, when connected to a 4K monitor, the Raspberry Pi 4B, 400 and CM4 will select a 30Hz refresh rate. Use this option to allow selection of 60Hz refresh rates.

Volume controllers
$_select[usb_volknob_on] $_select[usb_volknob_off]
Activates the triggerhappy key mapping daemon for USB volume knobs or other USB input devices. Refer to /etc/triggerhappy/triggers.d/media.conf for key/event mappings.
$_select[rotaryenc_on] $_select[rotaryenc_off]
Enable a rotary encoder device to control MPD volume.
Encoder settings (positional)
- Poll interval: Number of ms sleep for the volume update polling loop. Default = 100.
- Speed factor: Threshold (diff betw last/current enc pos) to determine whether to use 1 step or volume_step steps. Default = 2.
- Volume_step: Number of steps to use when knob turns at fast rate. Default = 3.
- Pin_A Pin_B: Broadcom SoC pin numbers. Default = 23 24.

$_select[lcdup_on] $_select[lcdup_off]
The LCD update daemon monitors the Metadata file /var/local/www/currentsong.txt. When changes are detected it runs a Python script named lcd-updater.py located in the /var/local/www/commandw/ directory.
NOTE: The Metadata file is automatically turned on when LCD updater is turned on.
